logging: a logging utility for ExtendScript¶

Logging is useful when debugging (nobody likes code interspersed with alert() statements). Logging is also beneficial in production settings — if something goes wrong, good logs will make it easy to know what happened and how to remedy the problem.

Extendables comes with a logging library, loosely inspired on the eponymous Python library.

Initialize a log using var log = new logging.Log("logfile.log");.

After that, you can log messages using log.debug(msg), log.info(msg), log.warning(msg), log.error(msg) and log.critical(msg).

All logs reside in the log directory under the Extendables root.

Log levels¶

level meaning
0 NOTSET
1 CRITICAL
2 ERROR
3 WARNING
4 INFO
5 DEBUG

Class Log¶

A log file object. Will resume an existing log if it exists, or make a new one if it doesn’t.

Examples¶

 var logging = require("logging");
 // all logs end up in extendables/log
 var log = new Log("example.log");
 try {
     throw new Error();
 } catch (error) {
     log.debug("Caught error, will log a debug message now");
     log.critical("Something happened: {error} ({env})", {'error': error, 'env': $.os});
}

Constructor¶

class Log(name[, log_level])¶
Arguments:
  • name (String) – the filename for this log file
  • log_level (String|Number) – the log level above which log messages don’t get written,

either a name (e.g. CRITICAL) or the actual log level. (Default: 4)
