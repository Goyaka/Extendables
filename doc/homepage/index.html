Extendables is a developers' framework for Adobe ExtendScript. Writing heavy-duty automations for InDesign, or anything more than just a throwaway script? This is for you.

  • Javascript goodies: we've added in as much features from Javascript 1.6+ as we could, like the .forEach() and .filter() methods on arrays.
  • Helpful shortcuts make your code less verbose. We've monkeypatched a bunch of additional methods on DOM objects like Document and XMLElement.
  • (Some) batteries included, like an HTTP library and a UI mini-framework.
  • Keep your sanity with support for logging and unit testing (using the excellent Jasmine framework). And we've got CommonJS-inspired namespaced modules. Because namespaces are one honking great idea.
  • Free as in freedom. Extendables comes with a very permissive MIT license. Feel free to use this framework in commercial closed-source projects â€” it's totally okay.

Code looks a little bit like this:

#include extendables.jsx
                
var http = require("http");
var placer = require("autoplacer");

var response = http.HttpRequest("http://www.example.org").toJSON();
response.forEach(function (story, i) {
    var page = current.doc.pages[i];
    var master = current.doc.masters[0];
    placer.place(story, page, master);
});

Check out the documentation for developers. Porting existing scripts is easy.

A few caveats. Extendables is still pretty young. Any Creative Suite scripter will find a lot to like, but the framework was created with InDesign scripting in mind. There's a ton of potential cool stuff that's not in there right now. But maybe, just maybe, even in its current incarnation, it'll make coding in ExtendScript fun again.
